# Commits and PRs Standard (AI Agents)

## Commits

- Keep commits scoped and coherent.
- Use clear commit messages describing intent.
- Avoid bundling unrelated changes.

## Before commit

- Review staged diff for secrets or sensitive values.
- Confirm included files are intentional.
- Ensure required checks have been run for the touched areas.

## Pull requests

- PR description should explain why the change exists and key impact areas.
- Mention any required follow-ups or known limitations.
- Ensure CI checks pass (including standards workflow).
